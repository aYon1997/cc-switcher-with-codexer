# Codex Relay 启动脚本使用文档
## 1. 脚本概述
本批处理脚本为 Windows 环境下 `codex-relay` 协议转换代理一键启动程序，作用为转发本地请求至上游 AI API 服务，提供本地兼容接口供 Codex 客户端调用。

## 2. 文件目录说明
### 2.1 程序二进制存放路径
工具根目录固定路径：C:\Users\.codex\tools\codex-relay\  (根据你电脑安装codex的实际路径进行调整)
- 可执行文件位置：`bin/codex-relay.exe`
- 完整程序路径：`C:\Users\WB-bw0017\.codex\tools\codex-relay\bin\codex-relay.exe`

### 2.2 脚本存放建议
将 `.bat` 启动脚本放置在任意方便执行的目录（如桌面、工具根目录），无强制路径要求。

## 3. 完整批处理源码
```batch
@echo off

chcp 65001 >nul

title Codex Relay - 协议转换代理

echo ========================================

echo      Codex Relay 启动中...

echo ========================================

echo.

echo 上游 API: http://xxx.xxx.x.xxx:xxxxx/v1

echo 本地监听: http://127.0.0.1:8787/v1

echo.

echo 提示：请勿关闭此窗口，否则 Codex 无法连接

echo       关闭 Codex 后可以按 Ctrl+C 停止本程序

echo.

echo ========================================

echo.

"C:\Users\WB-bw0017\.codex\tools\codex-relay\bin\codex-relay.exe" ^
  --upstream "http://xxx.xxx.x.xxx:xxxxx/v1" ^
  --api-key "sk-xxxxxxxxxxxxxxxxxxxxxxxxx" ^
  --port 8787

pause
```

## 4. 启动参数说明
执行 `codex-relay.exe` 时支持以下核心入参：

| 参数 | 作用 | 示例值 | 敏感标识 |
|------|------|--------|----------|
| `--upstream` | 上游远端API服务地址 | `http://192.168.1.244:13000/v1` | 非敏感 |
| `--api-key` | 上游接口鉴权密钥 | `**` | 高度敏感，必须隐藏 |
| `--port` | 本地代理监听端口 | `8787` | 非敏感 |

### 参数补充说明
1. `--upstream`
    后端大模型服务接口地址，根据实际部署的服务IP、端口修改。
2. `--api-key`
    上游服务身份凭证，**禁止明文泄露**，文档统一使用 `**` 替代真实密钥；
    替换脚本时填入自身有效密钥。
3. `--port`
    本地开放端口，Codex 客户端连接地址为 `http://127.0.0.1:{port}/v1`。

## 5. 使用步骤
1. 确认工具目录完整
    检查路径 `C:\Users\WB-bw0017\.codex\tools\codex-relay\bin\` 下存在 `codex-relay.exe`，缺失则重新部署工具包。
2. 修改配置（按需）
    - 修改 `--upstream` 后的上游服务地址
    - 将 `--api-key` 后引号内替换为自身有效密钥
    - 修改 `--port` 可更换本地监听端口
3. 执行脚本
    双击 `.bat` 文件运行，控制台自动切换 UTF-8 编码，打印启动信息。
4. 客户端连接
    Codex 客户端接口地址填写：`http://127.0.0.1:8787/v1`
5. 停止服务
    - 正常停止：控制台内按下 `Ctrl + C`
    - 强制停止：直接关闭CMD窗口

## 6. 重要注意事项
1. 密钥安全
    `api-key` 属于敏感凭证，对外分享脚本、文档时必须替换为 `**`，严禁明文暴露；
2. 窗口常驻
    代理运行期间不可关闭CMD窗口，关闭后本地转发服务立即中断；
3. 端口占用
    若 `8787` 端口被其他程序占用，修改 `--port` 参数更换未占用端口；
4. 编码设置
    `chcp 65001` 用于解决中文日志乱码，不可删除该行；
5. 路径修改
    若本机用户名变更，需要同步修改 `C:\Users\.codex\tools\...` 中的用户名部分。
